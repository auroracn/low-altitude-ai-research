# 个人研究建议

## 如何选题

### 选题原则
1. **兴趣驱动**：选择你最感兴趣的方向，保持长期动力
2. **能力匹配**：选择与当前技能水平匹配的课题
3. **价值导向**：选择有实际应用价值或学术价值的课题
4. **资源可及**：确保有数据、工具、硬件等资源支持

### 选题流程
```
了解全貌 → 确定方向 → 文献调研 → 发现问题 → 确定课题
```

### 推荐切入方向

#### 如果你擅长编程
- 目标检测模型训练与优化
- 路径规划算法实现与对比
- 仿真环境搭建与测试

#### 如果你擅长数据分析
- 低空经济市场数据分析
- 飞行数据分析与可视化
- 气象数据对飞行安全的影响

#### 如果你擅长写作/传播
- 行业研究报告撰写
- 技术科普短视频制作
- 学习笔记和知识总结

## 如何切入

### 第一步：建立全局认知
- 通读本知识库 `01-行业概览/`
- 了解行业背景、产业链、政策环境
- 时间：1-2 周

### 第二步：确定学习方向
- 阅读 `02-研究方向/` 的各个文件
- 选择 1-2 个最感兴趣的方向
- 时间：1 周

### 第三步：系统学习
- 按 `03-学习路径/` 分阶段学习
- 每个阶段设定明确目标
- 时间：按计划执行

### 第四步：动手实践
- 选择 `02-研究方向/推荐课题清单.md` 中的课题
- 从入门级课题开始
- 时间：1-3 个月/课题

### 第五步：知识转化
- 每个课题完成后提炼知识卡片
- 生成短视频脚本
- 持续积累内容

## 学习方法

### 费曼学习法
1. 学习一个概念
2. 用自己的话解释给别人听
3. 发现解释不清的地方，回去重新学习
4. 简化语言，让非专业人士也能理解

### 项目驱动学习
1. 设定一个具体项目目标
2. 学习完成项目所需的知识
3. 边学边做，遇到问题解决问题
4. 项目完成时，知识也掌握了

### 知识卡片法
1. 每个知识点提炼为一张卡片
2. 卡片包含：核心概念、关键要点、金句、案例
3. 定期回顾和更新卡片
4. 卡片可直接用于短视频脚本

## 避免的坑

1. **贪多求全**：不要同时学习太多方向，聚焦 1-2 个
2. **只看不练**：必须动手写代码、做实验
3. **闭门造车**：加入社区，与同行交流
4. **急于求成**：这是一个长期过程，保持耐心
5. **忽视基础**：AI 和编程基础要扎实
